## What now works

- Boot/media chain through live initramfs swap (Stage 1).
- Build-time dashboard HTTP + HTTPS readiness probes (`/readyz` over TLS).
- Supervised first-boot dashboard path (busybox-sh, respawn loop).
- TLS config + runtime prep for opt-in LAN HTTPS (`VYNTRIO_LAN_BIND_IP`).
- Authenticated appliance overview, settings, and **storage inventory** views.
- `GET /api/v1/storage/disks` end-to-end from UI (session + RBAC).
- Public landing honestly describes local-first flow, TLS opt-in, read-only storage inventory.

## Structurally present but not proven on boot

- Dashboard reachable after real firmware boot.
- First-boot owner creation on live runtime.
- Storage inventory on booted guest (API works on dev host; not boot-proven).
- LAN HTTPS reachability (requires operator env + booted network).
